# Archived Planning Documents

This directory contains the original planning documents created during the initial development phases of gh-tui. These have been superseded by the consolidated **PLAN.md** and **CHANGELOG.md** in the project root.

## Files

### BUILD_PLAN.md
Original comprehensive build plan created during project inception. Contains:
- Step-by-step implementation guide
- Architecture decisions
- Code examples and patterns
- Original timeline estimates

**Status**: Superseded by CHANGELOG.md (for completed items) and PLAN.md (for remaining work)

### NEXT_STEPS.md
Session-by-session TODO list and troubleshooting guide. Contains:
- Immediate next steps after Session 1
- Common compilation errors and fixes
- Testing checklist
- Debug commands

**Status**: Most tasks completed. Remaining work in PLAN.md

### ENHANCEMENTS.md
Comprehensive brainstorming document of future features. Contains:
- Interactive actions for each tab
- New view ideas (Notifications, Stars, etc.)
- UI enhancements
- Search/filter specifications
- Configuration options
- Integration features

**Status**: Features cherry-picked into PLAN.md Phase 6-7

### INTEGRATION_PLAN.md
Plan for integrating features from existing projects (tkan + TFE). Contains:
- tkan: Kanban board, drag & drop patterns
- TFE: Tree view, context menus, git operations
- Integration strategies
- Code porting guidelines

**Status**: Incorporated into PLAN.md Phase 2, 4, and 5

### PLUGIN_MANAGER_PLAN.md
Detailed specification for plugin management system. Contains:
- Tree view architecture (from TFE)
- Plugin detection and management
- Gist-based plugin distribution
- Installation/enable/disable workflows
- Phase-by-phase implementation plan

**Status**: Incorporated into PLAN.md Phase 2-3

## Using These Documents

- Comprehensive feature brainstorming
- Reference during implementation of planned features
